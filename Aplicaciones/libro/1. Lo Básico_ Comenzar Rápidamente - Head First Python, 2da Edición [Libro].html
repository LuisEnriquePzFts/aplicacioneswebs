"IDLE" se refiere al IDE, que toma su código de Python y lo ejecuta a través del intérprete. Es el intérprete quien hace todo el trabajo real aquí.

¿Qué pasó con mis llaves?

Si está acostumbrado a un lenguaje de programación que usa llaves ( {y }) para delimitar bloques de código, encontrar bloques en Python por primera vez puede ser desorientador, ya que Python no usa llaves para este propósito. Python usa indentación para demarcar un bloque de código, que los programadores de Python prefieren llamar suite en lugar de bloque (solo para mezclar un poco las cosas).

No es que las llaves no tengan uso en Python. Lo hacen, pero, como veremos en el Capítulo 3, las llaves duras tienen más que ver con delimitar los datos que con los conjuntos delimitadores (es decir, bloques ) de código.

En lugar de referirse a un "bloque" de código, los programadores de Python usan la palabra "suite". Ambos nombres se usan en la práctica, pero los Python prefieren "suite".

Las suites dentro de cualquier programa de Python son fáciles de detectar, ya que siempre están sangradas. Esto ayuda a su cerebro a identificar rápidamente las suites cuando lee el código. La otra pista visual para que usted pueda tener en cuenta es el carácter de dos puntos ( :), que se utiliza para introducir una suite que está asociada con ninguna de las sentencias de control de Python (como if, else, for, y similares). Verá muchos ejemplos de este uso a medida que avanza en este libro.

Confusión de importación

Esa es una gran pregunta.

Para que quede claro, no hay dos mecanismos de importación en Python, ya que solo hay una import declaración. Sin embargo, la importdeclaración se puede usar de dos maneras .

El primero, que inicialmente vimos en nuestro programa de ejemplo, importa una función nombrada en el espacio de nombres de nuestro programa , que luego nos permite invocar la función según sea necesario sin tener que volver a vincular la función al módulo importado. Dicho esto, vamos a esperar hasta un capítulo posterior para explorar los espacios de nombres en detalle).

En nuestro programa de ejemplo, usamos la primera técnica de importación, luego invocamos la datetimefunción como datetime(), no como datetime.datetime().

La segunda forma de usarlo importes importar el módulo, como lo hicimos al experimentar con el timemódulo. Cuando importamos de esta manera, tenemos que usar la sintaxis de notación de puntos para acceder a la funcionalidad del módulo, como lo hicimos con time.sleep().

No, no hay. No es por accidente que el código de Python sea fácil de leer: el lenguaje fue diseñado con ese objetivo específico en mente. Guido van Rossum , el creador del lenguaje, quería crear una poderosa herramienta de programación que produjera código que fuera fácil de mantener, lo que significaba que el código creado en Python también debía ser fácil de leer.

¿La sangría te está volviendo loco?

La sangría toma tiempo para acostumbrarse.

No te preocupes Todo el mundo que viene a Python desde un "lenguaje con rizado" lucha al principio con la sangría . Pero se pone mejor. Después de uno o dos días de trabajar con Python, apenas notará que está sangrando sus suites.

Un problema que algunos programadores tienen con la sangría se produce cuando mezclan pestañas con espacios . Debido a la forma en que el intérprete cuenta el espacio en blanco , esto puede generar problemas, ya que el código "se ve bien" pero se niega a ejecutarse. Esto es frustrante cuando comienzas con Python.

Nuestro consejo: no mezcle pestañas con espacios en su código Python .

De hecho, iríamos más allá y le aconsejaríamos que configure su editor para reemplazar un toque de la tecla Tab con cuatro espacios (y mientras lo hace, eliminará automáticamente cualquier espacio en blanco al final). Esta es la convención bien establecida entre muchos programadores de Python, y usted debe seguirla también. Tendremos más que decir sobre cómo tratar la sangría al final de este capítulo.

Arranque, parada y escalonamiento

Como rangeno es el único lugar donde te encontrarás con start , stop y step , tomémonos un momento para describir lo que significa cada uno de ellos, antes de ver algunos ejemplos representativos (en la página siguiente):

  1. El valor START le permite controlar desde DONDE comienza el rango.

    Hasta ahora, hemos utilizado la versión de argumento único range, que, según la documentación, espera que se proporcione un valor para detener . Cuando no se proporciona ningún otro valor, el valor rangepredeterminado es usar 0como el valor de inicio , pero puede establecerlo en un valor de su elección. Cuando lo haga, debe proporcionar un valor para detenerse . De esta forma, se rangeconvierte en una invocación de múltiples argumentos.

  2. El valor STOP le permite controlar CUANDO termina el rango.

    Ya hemos visto esto en uso cuando invocamos range(5)en nuestro código. Tenga en cuenta que el rango que se genera no contiene la parada de valor, por lo que es un caso de arriba-a-pero-no-incluyendo la parada .

  3. El valor de STEP le permite controlar CÓMO se genera el rango.

    Al especificar los valores de inicio y detención , también puede (opcionalmente) especificar un valor para el paso . Por defecto, el paso de valor es 1, y esto le dice rangea generar cada valor con un paso de 1; es decir, 0, 1, 2, 3, 4, y así sucesivamente. Puede establecer el paso a cualquier valor para ajustar la zancada. También puede establecer el paso a un valor negativo para ajustar la dirección del rango generado.
